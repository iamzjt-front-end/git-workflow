# 🎉 Git Wrapped - 年度编程回顾

> 将你的Git活动转化为精美的年度报告，让编程成就可视化

## 📋 功能概述

Git Wrapped 是一个受 Spotify Wrapped 启发的功能，它分析你的Git提交历史，生成一份精美的年度编程报告。这不仅是对过去一年编程工作的总结，更是对个人成长的庆祝。

## 🎯 核心功能

### 基础命令
```bash
gw wrapped              # 生成当前年份报告
gw wrapped 2024         # 生成指定年份报告
gw wrapped --share      # 生成可分享的图片版本
gw wrapped --format png # 指定输出格式 (png/svg/pdf)
gw wrapped --theme dark # 指定主题风格
```

### 高级选项
```bash
gw wrapped --repo ./my-project    # 分析特定仓库
gw wrapped --author "张三"        # 分析特定作者
gw wrapped --range 2024-01-01..2024-12-31  # 自定义时间范围
gw wrapped --compare 2023         # 与去年对比
gw wrapped --team                  # 生成团队报告
```

## 📊 报告内容

### 1. 📈 基础统计
- **总提交数**: 今年的总提交次数
- **活跃天数**: 有提交活动的天数
- **最长连击**: 连续提交的最长天数
- **代码行数**: 新增/删除的代码行数统计
- **文件变更**: 修改的文件总数

### 2. 🏆 年度亮点
- **最忙碌的月份**: 提交最多的月份
- **最活跃时间**: 最常提交的时间段
- **最大单日提交**: 单日最多提交次数
- **里程碑时刻**: 重要的提交节点
- **项目贡献**: 贡献最多的项目

### 3. 💻 技术分析
- **编程语言分布**: 各语言使用占比
- **文件类型统计**: 最常修改的文件类型
- **提交类型分析**: feat/fix/docs等类型分布
- **项目活跃度**: 各项目的贡献分布

### 4. ⏰ 时间模式
- **编程习惯**: 早鸟 vs 夜猫子分析
- **工作日 vs 周末**: 工作时间分布
- **月度趋势**: 全年活跃度变化曲线
- **季节性模式**: 不同季节的编程活跃度

### 5. 🎨 个性化洞察
- **编程风格**: 基于提交信息的风格分析
- **专注度**: 单次提交的平均文件数
- **完美主义指数**: 小提交 vs 大提交的比例
- **探索精神**: 新项目/新语言的尝试次数

## 🛠️ 技术实现

### 数据收集
```typescript
interface GitWrappedData {
  year: number;
  totalCommits: number;
  activeDays: number;
  longestStreak: number;
  linesAdded: number;
  linesDeleted: number;
  languages: LanguageStats[];
  repositories: RepoStats[];
  timePatterns: TimePattern;
  commitTypes: CommitTypeStats;
  achievements: Achievement[];
}
```

### 分析算法
1. **Git日志解析**: 使用`git log`命令获取提交历史
2. **语言检测**: 分析文件扩展名和Git属性
3. **时间模式**: 统计提交时间分布
4. **趋势分析**: 计算月度、季度变化
5. **成就系统**: 基于规则检测特殊成就

### 图片生成
- **Canvas API**: 使用Node.js的canvas库
- **SVG生成**: 矢量图形支持
- **模板系统**: 可配置的设计模板
- **字体支持**: 中英文字体兼容
